## 前言

本文可能更适合需要更高需求的学生（例如需要调用某个包），对于普通的学生而言，默认情况下的Visual Studio已经够用了。

本文将在两类操作系统上进行试验，其中可能会有间接宣传。

## Windows

### 使用msvc（Visual Studio）工具链

#### 安装

直接前往[官网](https://visualstudio.microsoft.com/)下载即可。

#### vcpkg

对于一些需要使用第三方库（尤其是**非**仅头文件库）的学生，我建议尝试一下`vcpkg`（反正也是本地编译）。

参考[安装](https://learn.microsoft.com/zh-cn/vcpkg/get_started/get-started?pivots=shell-powershell)进行安装（可能需要先下载
```git```）。
然后在Visual Studio进行配置即可。

### 使用GNU工具链

#### 安装

笔者推荐直接使用`msys2`环境而不是单独下载编译器等内容，因为其中附带了一些工具，可以直接进行使用。

笔者未尝试过直接在官网下载，相反，可以在开源镜像站进行下载，速度*可能*会比官网快些。

#### 使用

- 选择

  可以根据自己的情况酌情使用下面的几个工具链，笔者推荐**UCRT64**的。

  ![msys_main](/img/07/cppenv/msys_main.png)

  为啥不推荐使用**MINGW64**的？因为这个链接的运行时库太老了，老到什么程度呢？甚至不能直接用`setlocale`将语言编码设置成UTF-8。
  **UCRT64**的直接就能链接到新的ucrt库，这个库在现在的微软官网是有文档的，直接查询相关函数就能搜到。

- 安装包

  msys2使用`pacmam`包管理器（和Arch
  Linux）的一样，因此如果想查阅命令的相关用法，可以参考[这里](https://wiki.archlinuxcn.org/wiki/Pacman)。
  
  不过需要用的命令可能比较少，以下是一些举例：

    - `pacman -Ss` 搜索包
    - `pacman -Sy 包名` 安装包
    - `pacman -Syu` 升级包
    - `pacman -R` 卸载包
  
  因为这些工具链是共用一个包管理的，所以在安装某个对应工具链的包时需要加上“前缀”，例如`mingw-w64-ucrt-x86_64`，搜索包时后面也会标明。

- 设置环境变量

  **特别警告：不要将msys的目录放进PATH中，会导致系统错乱！**

  这一步是可选的，如果希望能调用到某个工具链的命令，可以将其的可执行目录加入到环境变量的`PATH`中。例如`C:\msys64\ucrt64\bin`。

- 编译第一个程序

  这边建议配合一个有插件功能的编辑器（例如`vscode`）或集成开发环境（例如`CLion`）使用。（最好配合语言服务器使用，一般使用的是`clangd`）

  编译程序首先需要编译器，可以使用`gcc`或者`clang`，按上述教程安装，然后尝试`gcc(或clang) 文件名 -o 输出文件`编译。

  如果需要使用构建程序，最好是直接在工具链中安装并使用工具链提供的，最好不要自行在官网下载。

#### 打包

其实我也不太会，不过笔者之前找了个deploy脚本，可以试试：

```bash
deplist=$( ldd $1 | awk '{if (match($3,"/ucrt64")){ print $3}}' )
cp -L -n $deplist $2
```
